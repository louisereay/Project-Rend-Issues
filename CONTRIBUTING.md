## Reporting Bugs

### Before Submitting a Bug Report

1. **Check the [Wiki](../../wiki)** - Your issue might be documented
2. **Search existing issues** - Avoid duplicates
3. **Gather information** - Prepare logs, screenshots, and environment details

### How to Submit a Good Bug Report

Use our [bug report template](../../issues/new?template=bug_report.md) and include:

- Clear, descriptive title
- Exact steps to reproduce
- Expected vs actual behavior
- Environment details (OS, Node.js version, etc.)
- Relevant logs or error messages
- Screenshots if applicable

## Suggesting Features

### Before Submitting a Feature Request

1. **Check existing features** - It might already exist
2. **Search issues** - Someone might have already suggested it
3. **Consider the scope** - Does it align with Project Rend's goals?

### How to Submit a Good Feature Request

Use our [feature request template](../../issues/new?template=feature_request.md) and include:

- Clear use case
- Problem it solves
- Proposed implementation (if you have ideas)
- Alternative solutions considered
- Potential impact on existing features
